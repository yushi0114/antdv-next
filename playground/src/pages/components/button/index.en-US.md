## When To Use {#when-to-use}

A button means an operation (or a series of operations). Clicking a button will trigger its corresponding business logic.

In Ant Design we provide 5 types of button.

- üîµ Primary button: used for the main action, there can be at most one primary button in a section.
- ‚ö™Ô∏è Default button: used for a series of actions without priority.
- üò∂ Dashed button: commonly used for adding more actions.
- üî§ Text button: used for the most secondary action.
- üîó Link button: used for external links.

And 4 other properties additionally.

- üî¥ `danger`: used for actions of risk, like deletion or authorization.
- üëª `ghost`: used in situations with complex background, home pages usually.
- üö´ `disabled`: used when actions are not available.
- üîÉ `loading`: adds a loading spinner in button, avoids multiple submits too.

## FAQ

### How to choose type and color & variant? {#faq-type-color-variant}

Type is essentially syntactic sugar for colors and variants. It internally provides a set of mapping relationships between colors and variants for the type. If both exist at the same time, the colors and variants will be used first.

```vue
<template>
  <a-button type="primary">
    click
  </a-button>
</template>
```

Equivalent

```vue
<template>
  <a-button color="primary" variant="solid">
    click
  </a-button>
</template>
```

### How to close the click wave effect? {#faq-close-wave-effect}

If you don't need this feature, you can set `disabled` of `wave` in [ConfigProvider](/components/config-provider#api) as `true`.

```vue
<template>
  <a-config-provider :wave="{ disabled: true }">
    <a-button>click</a-button>
  </a-config-provider>
</template>
```
